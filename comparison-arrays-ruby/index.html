Comparison of arrays in Ruby

# 28 July 2015 on Ruby

This morning I was working with arrays on Ruby 2.1.2. I have compiled for you different ways to compare arrays with others arrays or values.

Comparing values

We can use include? to ask the array if it includes a value in its elements. This method return true or false

a = ["a","b","c"]
a.include? "a"         # true

There an alternative, the index method. If the array contains the value, this method will return the index of value inside array. If the value doesn't exist, it will return nil.

a = ["a","b","c"]
a.index "z"         # nil

First method is better, but index can be useful in some situations.

Comparing two arrays

When we want to compare two arrays the method to use isn't direct. To compare them we can use basic and binary operators.

Coincidence of some value

A way to check if a some value of an array is contained in other is to use AND operator. This operator get the intersection of two arrays, so if there are a coincidence we will get a non empty array.

A = ["a","b","c"]
B = ["c","d","e"]
C = A & B
# C = ["c"]

We can use this in conditionals:

C.empty?
# If true, there are a coincidence of values between A and B

Inclusion of arrays

To do this job, the best operator is subtract. If an array (A) is contained in other (B), the result will be an empty array:

A = [1, 2]
B = [1, 2, 3]

(A - B).empty?
# true, so A is contained on B

An alternative is to use AND operator. If the intersection has same size of B, A is included:

(A & B).size == B.size
# true, so A is included in B
